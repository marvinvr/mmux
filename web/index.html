## what you get

┌─ per-directory & persistent ─┐

per-directory & persistent

one mmux per directory, kept alive inside a tmux session. detach, drop ssh, reattach — it's all still there.

└──────────────────────────────┘
┌─ agents on demand ─┐

agents on demand

spawn claude, codex, whatever. each runs in its own pane, started and restarted straight from the sidebar.

└────────────────────┘
┌─ processes you watch ─┐

processes you watch

start, stop and tail your dev server and tasks without ever leaving the multiplexer.

└──────────────────────┘
┌─ attention, caught ─┐

attention, caught

a bell or a notification escape becomes a sidebar dot and a real desktop notification. even over ssh.

└─────────────────────┘
┌─ linked projects ─┐

linked projects

group sibling clones into one sidebar, each its own section; the panel follows whichever project is active.

└───────────────────┘
┌─ one binary, any terminal ─┐

one binary, any terminal

a single rust binary. it runs anywhere a terminal does.

## how it works 

  your terminal  (local · ssh · tmux client)
        │
        ▼
  ┌─────────────────────────────────────────────┐
  │  tmux session   (one per directory · hidden) │   ← survives detach / disconnect
  │  ┌───────────────────────────────────────┐   │
  │  │  mmux  (ratatui TUI)                   │   │
  │  │   sidebar  │   main pane   │  panel    │   │
  │  │  agents    │  focused      │  lazygit  │   │
  │  │  terminals │  program's    │  (git)    │   │
  │  │  processes │  live screen  │           │   │
  │  └───────────────────────────────────────┘   │
  └─────────────────────────────────────────────┘
   every pane = a real PTY + a vt100 parser
  • the TUI runs inside a per-directory tmux session, so closing the terminal or losing ssh never kills it.
  • each agent, terminal and process is one pty-backed pane behind a single unified lifecycle.
  • the bell and notification escapes are captured and turned into desktop notifications.

## install

homebrew
brew install marvinvr/mmux/mmux
cargo
cargo install mmux
from source
git clone https://github.com/marvinvr/mmux && cd mmux && cargo install --path .

then just run mmux (in a dir of your choice)
