# Module Map

A file-by-file index of the source tree. For the concepts behind it, read
[Architecture](06-architecture.md) first. Keep this map current — see the
[documentation covenant](08-contributing.md#maintaining-this-documentation).

The binary has two halves: a thin CLI/tmux outer shell, and the `app/` TUI it launches.

## Outer Shell (`src/`)

| File | Responsibility |
| --- | --- |
| `main.rs` | Entry point — declares the module tree and calls `cli::run()`. |
| `cli.rs` | Argument dispatch and the non-TUI subcommands: `--help`/`--version`, `init`, `agents` (terminal agent manager → `wizard::run_agents`), `check`, `docs`/`doc`, `attach`/`a`, and the inner-vs-outer (`MMUX_INNER`/`--inner`) split. Also holds the embedded `mmux docs` setup guide. |
| `tmux.rs` | All tmux interaction: attach-or-create the per-directory session (`launch`/`launch_in`), configure it invisible, `detach`, the recents log (`~/.mmux/history`), the `mmux attach` picker, and `client_supports_sixel` (does the attached terminal do sixel? — gates the real-pixel image preview). Session name = hash of the canonical dir. |
| `wizard.rs` | Interactive `mmux init` (and the first-run path): picks agents via an inline checkbox picker (`select_agents` drives an [`agentmgr::AgentManager`](#outer-shell-src) in raw mode, redrawing the rows in place — no full-screen takeover), then asks start commands and linked projects and writes commented YAML. Agents seed a fresh global config when none exists, else go local. Also `run_agents` (the `mmux agents` command: the same picker over the existing global config). Pure builders (`split_command` + the `build_*` YAML formatters) are unit-tested. |
| `agentmgr.rs` | Shared, front-end-agnostic state for the built-in agent harnesses: a `Row` per [`config::PRESETS`](#outer-shell-src) entry (`enabled` / danger via `args` / `installed`, probed with `on_path`) plus any non-preset agents preserved in `custom`; toggle helpers + `drafts()` (the full list to write). `new()` reads the current global config (in-TUI popup + `mmux agents`), `fresh()` pre-checks installed harnesses (`mmux init`). Drives all three front-ends: the sidebar-`a` popup, the terminal picker, and the init step. Unit-tested. |
| `config/mod.rs` | The typed `mmux.yaml` + `~/.mmux/config.yaml` schema, load & merge (project over global via `deep_merge`; agents/processes by name), and `load_workspace` (root + linked projects, one level deep, path-deduped, capped at 8). Owns the built-in agent presets (`AgentPreset` / `PRESETS` / `preset_by_name`), shared by the wizard and the in-TUI agent manager, and the `ProcessDraft` / `AgentDraft` the forms hand to the writers. Plus `split_command` / `join_command` (inverse, to pre-fill the edit form) and the path helpers (`global_config_path` / `relative_path`). Re-exports everything in [`yaml`](#outer-shell-src) so the public API stays `crate::config::*`. Clean — touch sparingly. |
| `config/yaml.rs` | The comment-preserving **raw-text** YAML editors used by the in-TUI forms — `append_process` / `append_linked_project` (splicing an item into a block, or seeding a documented `scaffold_project_file` when the target file is missing or blank), `replace_process` / `remove_process` (sharing `named_item_span`) for editing/deleting a process by name, and `write_agents` (rewrites the whole global `agents:` block via `replace_agents_block`, scaffolding a fresh global file when absent; `global_agents` reads it back for the manager). A `splice_lines` primitive + `append_block` helper back the four editors; `write_starter` + the `STARTER` const and the shared onboarding-comment consts (reused by the wizard) round it out. Splices raw text rather than round-tripping serde, which would strip every comment. Clean — touch sparingly. |
| `pane.rs` | One PTY-backed pane: `portable-pty` + a `vt100` parser on a reader thread, a writer thread draining an input channel, a reaper thread. A `Callbacks` impl captures the OSC title, bell, and notification OSCs (9/777/99); `contents_block` stitches scrollback text for copy; `cwd()` reads the child's live working directory (`/proc` on Linux, `proc_pidinfo` on macOS) so a `cd` survives a restore; `wheel_input`/`mouse_input` (sharing the `mouse_seq` encoder) translate the wheel and clicks/drags into the inner program's negotiated mouse protocol. Clean — touch sparingly. |
| `notify.rs` | Pure formatting of the desktop-notification escape (OSC 9/777/bell), the tmux passthrough wrapping, and the `command` fallback. Unit-tested. |
| `update.rs` | Background self-update for the two managed install kinds (`install_kind`): Homebrew and the `mmux.org/install.sh` native binary. A cheap version check — first a local "did a sibling already upgrade the on-disk binary?" test, then follow the GitHub `releases/latest` redirect and compare versions (unit-tested) — gates staging. A self-managed install downloads the tarball and atomically swaps the binary in place; a brew install runs a (confirmed) `brew upgrade`. Both report over an `mpsc` channel; `exec_restart` applies a staged update in place. Inert for unmanaged/dev builds. |
| `agent.rs` | Hardcoded, no-config resume support for the two preset agents, detected by command basename: Claude (mmux *owns* the session id — `--session-id` / `--resume`, so each agent keeps its own thread and several never get mixed up) and Codex (`codex resume`). `sessions_for` lets a fresh Codex agent discover the session it just created from the tool's transcripts (newest for the cwd). Mints UUIDs without a crate. Unit-tested. |
| `restore.rs` | The per-workspace [restore-state](06-architecture.md#session-restore) file (`~/.mmux/state/<session-hash>.yaml`): the serde model (agents/terminals + their cwd and Claude/Codex session id) and best-effort save/load. Lets a reopen (quit/crash/update) bring everything back. |
| `clipboard.rs` | `copy(text)` — writes an OSC 52 escape to stdout **and** pipes to a local helper (`pbcopy`/`wl-copy`/`xclip`/`xsel`), with a hand-rolled base64 encoder. |
| `open.rs` | `url(url)` — open a link in the default browser, best-effort, via the platform opener (`open`/`xdg-open`/`start`). Fire-and-forget on a throwaway thread; talks to the OS directly, so the tmux jail doesn't get in the way. Backs the clickable [About-card and pane-content links](03-usage.md#opening-links). |
| `git.rs` | Stateless, synchronous wrappers over the `git` CLI for the native git panel: parse `status`/`log`/`branch`; stage/discard/commit/switch/pull/push/stash; the commit ops `show`/`revert`/`soft_reset`/`commit_message`; `diff` one file for the preview; build the flattened changed-files tree. Errors come back as plain strings. |

## The TUI (`src/app/`)

One inherent `impl App`, split across files. `mod.rs` defines the struct; the others each add
`impl App { … }` for their concern.

| File | Responsibility |
| --- | --- |
| `mod.rs` | The `App` struct (all state) and the `Project` struct (per-project cfg / counters / git panel) + `new()` (which restores saved sessions on startup) + the `run()`/`run_loop()` event loop (a final state save as it exits, then the apply-update re-exec) + `emit_sixel` (paints a pending image-preview sixel on top of each frame, tracking `last_sixel` so a static picture isn't re-sent) + `tick()` (follow-active, git job drain, panel refresh, exited-agent/terminal pruning, self-update polling, restore-state save) + the `UpdateState` badge state machine + notification drain/emit + the `flash`/`flash_result`/`flash_err` footer-note helpers + focus/resize helpers. |
| `session.rs` | The unified pane-backed model: `Session` / `Recipe` / `Kind` / `Status`. One `spawn()`/`stop()`/`kill()` lifecycle for agents, terminals, and processes; `spawn` appends any Claude/Codex resume flags from the optional `agent` slot. `Recipe` builders: `agent`/`process`/`shell`/`editor`. A process's optional `stop:` teardown rides on `Session.stop`, exposed as a ready-to-run `stop_command()` (a `sh -c` in the recipe's cwd) the lifecycle fires when the process stops. |
| `nav.rs` | The sidebar nav list (`build_nav`) + the positional selection cursor (`sel`) + resolvers: `project_of`/`current_nav`/`pane_at`/`move_sel`/`jump_project`/`focus_project`/`select_session`/`focused_pane`. |
| `lifecycle.rs` | Sidebar-driven actions: `spawn_agent` (tags Claude/Codex with a resume slot) / `spawn_terminal`, `open_new_process`/`finish_new_process` (append or, in edit mode, `replace_process`), `edit_selected`/`delete_selected`/`delete_process` (the process edit & confirm-delete flow), `open_agent_manager`/`apply_agent_manager` (the sidebar-`a` popup — write the global agents + reload), `open_picker`/`open_in_editor`, `open_link_browser`/`link_project` (grow the workspace with a linked project), `activate`, `do_start`/`do_stop`/`do_restart`, `close_session` (lands the cursor on the row that slides into the freed slot — the next sibling of the same kind — falling back to the previous sibling for the section's last row, or the section launcher when it empties) / `close_named_session` (the deferred, by-name half of the running-agent/terminal close confirmation) / `prune_exited`, and `reload` (live config re-read & reconcile — a running process whose command changed is restarted). Stopping a running process fires its `stop:` teardown (`run_stop_command`, backgrounded); `run_stop_commands_on_quit` runs them all and waits, called from `run()` as mmux exits. |
| `persist.rs` | [Session restore](06-architecture.md#session-restore): `save_state` snapshots the live agents/terminals (live cwd + `refresh_agent_ids`, which binds each agent to its own resume id — Claude's owned, a fresh Codex's discovered — so they don't get mixed up) on structural change and as the loop exits; `restore_sessions` rebuilds them on startup (Claude/Codex resumed, shells at their cwd) and bumps the name counters. Backed by `restore.rs` / `agent.rs`. |
| `input.rs` | Core input routing. `on_key` (an open modal → `overlay_key` in [`overlay`](#the-tui-srcapp), else global `Ctrl+P` → global `Ctrl-b` leader (`leader_command`) → focus dispatch), `key_sidebar`/`key_pane`/`key_git`, `on_mouse` (click/drag/wheel routing), `forward_mouse` (hand the event to a mouse-tracking inner program unless Shift bypasses), `on_paste`, the drag-select-to-clipboard machinery — over a live vt100 pane (`SelTarget::Main`, block cells) **and** over the diff pager (`SelTarget::Diff`, whose `diff_cell`/`copy_diff_selection` map to `(line, content column)` so the gutter number and `+`/`-` sign fall outside the selection) — footer-button actions, opening a clicked link (`url_at`/`url_under`/`open_url` — a plain, non-drag click on a URL in a pane, or on an About-card link while a modal captures the mouse), and the `hit`/`button_code` helpers. `Selection`/`SelTarget` live here. |
| `keymap.rs` | `encode_key`: pure crossterm-key → PTY-byte translation. Unit-tested. |
| `git.rs` | The native git panel: `GitPanel` (refresh, three-box cursor/section nav, stage/commit/discard/stash, backgrounded pull/push via `RemoteOp`, and the Commits ops — show diff, copy hash/message, revert, soft-reset) with `JobDone`/`Section`, and the `impl App` git-action methods — including the diff-lifecycle bridges (`git_open_diff`/`git_show_commit`/`git_preview_follow`/`diff_upkeep`/`diff_scroll`/`clear_diff`) that drive the [`diff`](#the-tui-srcapp) preview. **Not** a pane. The `Overlay` modal enums now live in [`overlay`](#the-tui-srcapp), the `DiffView` model in [`diff`](#the-tui-srcapp). |
| `diff.rs` | The read-only diff preview shown in the centre pane. The `DiffView` model + the shared `parse_diff`, which turns a `git diff` **or** `git show` (a selected commit, carried as a `CommitRef`) into `DiffLine`s (`DiffKind`, incl. a `File` divider for multi-file commits + new-file line number + syntax-highlighted spans via [`highlight`](#the-tui-srcapp)); and, for a working-file image, the image path (`PreviewImage`/`HalfCell` — `is_image_path`/`rasterize`/`encode_sixel`) that decodes a changed picture and caches it per size both as a `sixel` string (real pixels) and a half-block grid (fallback). Built and kept in sync by [`git`](#the-tui-srcapp), drawn by [`view::pane`](#view-srcappview). **Not** a pane. |
| `overlay.rs` | Overlay **state + input**: the `Overlay`/`PromptKind`/`Confirmed` modal enums (`Overlay` spans the commit/new-branch `Prompt`, the yes/no `Confirm`, the `Picker`, the `NewProcess` form, the `LinkProject` browser, the `Agents` manager popup, and the stateless `About` card) and their `impl App` key handlers while one is open — `overlay_key`/`overlay_submit`/`overlay_confirm` plus the per-modal `about_key`/`agentmgr_key`/`procform_key`/`procform_advance`/`linkbrowse_key`. Rendered by [`view::overlay`](#view-srcappview). |
| `highlight.rs` | Syntax highlighting for the diff preview: a lazily-loaded `syntect` `SyntaxSet`/`Theme` (pure-Rust `fancy-regex` backend, `base16-ocean.dark`) behind a `OnceLock`, and a per-file `Highlighter` that colours code **one line at a time** into ratatui RGB spans (stateless — a diff isn't contiguous source; oversized diffs fall back to plain). |
| `picker.rs` | The `Ctrl+P` fuzzy file picker: list files with the `ignore` crate (ripgrep's walking engine, in-process — no external `rg`; `.gitignore` deliberately not honoured, heavy build/artifact dirs pruned) and rank them. Fuzzy score is unit-tested. |
| `procform.rs` | State for the process guided form (Name → Command → Cwd → Stop → Review) with an autostart toggle and per-field validation. Shared by add (`ProcForm::new`) and edit (`ProcForm::edit`, pre-filled; `edit: Some(original_name)` routes the write to `replace_process` and exempts the entry from the duplicate-name check). |
| `linkbrowse.rs` | State for the `+ Link another project` directory browser: a fork-free filesystem walk (subdirs, filtered) with cheap per-dir previews (`.git`/`mmux.yaml` checks, branch from `.git/HEAD`), live filter + cursor, and the path the highlighted dir would be linked as. Applied by `lifecycle::link_project`. |

### View (`src/app/view/`)

| File | Responsibility |
| --- | --- |
| `mod.rs` | `draw()`: the responsive layout split, the per-frame `Regions` hit-rects, and the footer — `footer_segments()` returns a left + a right-aligned cluster of clickable shortcut chips (`FooterAction`/`Seg`, laid out by `layout_segs`), which in compact mode become the `menu`/`git`/`✕ close` toggle in the two bottom corners. `COMPACT_W` / `MAIN_MIN` live here. |
| `sidebar.rs` | `render_sidebar`: per-project boxes (or a single drawer) filling the body above a bottom-pinned link box (`split_link_box`; the active box expands via `box_heights`), the `AGENTS`/`TERMINAL`/`PROCESSES` (+ compact `GIT`) sections, one styled row per nav entry via `nav_row()`, and the standalone `Projects`-titled `+ Link another project` box (`render_link_box`; the row is drawn by `nav_row` like a `+ New …` launcher) — a nav row (`Nav::Link`), so it's arrow-navigable and highlights when selected. |
| `pane.rs` | `render_main` / `render_right` (share `render_screen`/`render_placeholder`), `main_title`, `placeholder_text`, the diff preview (`render_diff`/`diff_line`/`diff_title`; `diff_line` draws the line-number gutter + sign + highlighted spans with a full-row add/del tint, and for an image it stashes a sixel for `emit_sixel` or paints the `render_image` half-block fallback), the focused-pane cursor placement, and the drag-selection overlays (`paint_selection` for the pane, `paint_diff_selection` for the diff — content columns only). `render_right` delegates the real panel to `view::git`. |
| `git.rs` | All git-**panel** rendering: the three bordered git boxes (Changes tree · Branches · the selectable, scrollable Commits history), drawn through a shared `render_box` helper with their row/tree helpers, returning the click-routing row maps (`GitRows`). The modal overlay renderers now live in [`overlay`](#view-srcappview). |
| `overlay.rs` | All modal rendering: `render_overlay` dispatches to the seven modal renderers — the commit/new-branch `render_prompt`, `render_confirm`, `render_picker`, the `render_procform`, `render_agentmgr` and `render_linkbrowse` browser, and `render_about` (the About card, drawn from `draw` so it can read live update state) — sharing the modal chrome (`modal_frame` over a `centered` rect) and helpers (`list_scroll`/`mmux_logo_lines`/`linkrow`/`preview_lines`/`field_line`). The overlay state + input lives in [`overlay`](#the-tui-srcapp). |
| `theme.rs` | Shared styling primitives: the named color constants (the `ATTN` accent + the `SPINNER` frames, the diff palette, `SELECTION_BG`/`ACTIVE_HEADER_BG`, the `FOCUS_BORDER`/`ACTIVE_BORDER`/`IDLE_BORDER` accents, the `FOOTER_*` bar) and `fill_row_bg` (paint a row-wide highlight), plus `status_style`/`status_label`/`badge`, `agent_glyph_style` (the glyph + attention-aware name color for agent/terminal rows), `header`, `project_header`, and `entry_line` (the universal sidebar row). Shared by the sidebar, git panel, and overlays. |

## Repository Layout (beyond `src/`)

| Path | What it is |
| --- | --- |
| `docs/` | This documentation — the canonical source of truth. |
| `README.md` | Short user-facing overview; points here. |
| `AGENTS.md` | Agent/contributor orientation + the documentation covenant. `CLAUDE.md` is a symlink to it. |
| `mmux.yaml` | mmux's own config (build/dev processes — open this repo *with* mmux). |
| `.github/workflows/` | CI/CD. `_build.yml` (reusable: build + smoke each target on its native-arch runner, stamped `MMUX_RELEASE=1`), `ci.yml` (every commit: lint/test/build+smoke/boot-test), `release.yml` (tag: staged prerelease → parallel `verify` → gated `promote` to latest + tap bump → issue on failure), `healthcheck.yml` (weekly live-channel check → auto-`yank` on rot), `yank.yml` (demote release + revert tap). `.github/scripts/` renders/pushes the formula; `.github/homebrew/mmux.rb` is the template. See [contributing](08-contributing.md#release). |
| `ci/` | Shared test scripts: `smoke.sh` (non-TTY subcommand checks, reused by every layer), `boot-test.sh` (best-effort TUI boot under a PTY), `verify-distros.sh` (run the installer inside distro containers to prove musl portability). |
| `web/install.sh` | The `curl \| sh` installer served at `mmux.org/install.sh`: detects OS/arch, downloads + checksum-verifies the matching release tarball, installs to `~/.local/bin`. |
| `web/` | The `mmux.org` static marketing site (plain HTML/CSS/JS). Has its own `README.md` and `DESIGN.md`; not compiled into the binary. |
| `dist/` | Output of the "Build (all platforms)" process — distributable binaries. |
